# fiction-analyze：网文拆解

拆解爆款小说，产出结构化分析报告，供写作时自动引用。

## 路由

按篇幅自动分流：
- ≥3 万字 → 长篇拆解管线
- <3 万字 → 短篇拆解管线

## 长篇拆解管线

### Stage 1：黄金三章分析

读取前 3 章内容，分析：
- 开篇钩子类型与效果
- 主角出场方式与角色定位
- 核心冲突引入节奏
- 爽点密度与分布
- 第一章字数与信息密度

产出快速预览报告，询问用户是否继续全量拆解。

### Stage 2-6：全量拆解（用户确认后继续）

- 逐章摘要（完整章节省略，只输出关键事件/情绪节点/钩子）
- 聚合分析：角色设计/剧情结构/设定体系/写作手法
- 设定与关系映射
- 汇总报告（含：风格特征、可借鉴模式、差异化建议、节奏图谱）

### 产出

```
拆文库/{书名}/
├─ 黄金三章分析.md
├─ 拆解报告.md       # 核心发现
├─ 风格特征.md       # 文风/节奏/技法
├─ 章节摘要/         # 逐章（或关键章节）摘要
├─ 角色分析.md       # 角色设计模式
├─ 剧情结构.md       # 结构段位/反转/节奏
└─ 写作手法.md       # 可借鉴技巧
```

## 短篇拆解管线

全量拆解，直接产出完整报告：
- 故事核、结构段位、情感线、反转设计
- 写作手法（节奏控制、情绪点设计、钩子密度）
- 共鸣层次分析
- 可借鉴模式 + 差异化建议

### 产出

```
拆文库/{书名}/
├─ 拆解报告.md
├─ 情节节点.md        # 情绪曲线 + 节奏标注
├─ 写作手法.md        # 可借鉴技巧
└─ _meta.json         # 元数据（字数/类型/对标价值评分）
```

## 自动引用规则

拆文结果写入 `拆文库/{书名}/`，写作时自动读取：
- fiction-write 的 context-agent 按"本回目标情绪"匹配最相关的拆文段落
- 对照参考技法而非抄袭原文字句

## 参考

拆解方法论参考：
- material-decomposition.md：素材拆解方法
- style-profile-generator.md / style-profile-protocol.md：风格特征生成
- deconstruction-notes.md：拆解笔记规范
- output-templates.md：输出模板

以上参考在对应阶段加载，不预读。
拆解完成后，报告会自动写入 `拆文库/` 目录。
如果拆解过程中涉及浏览器操作（如登录平台查看章节），优先使用 browser-cdp 技能。
拆解仅作模式分析用途，不存储原文内容。
