# novel-switch：多书切换

支持在同一项目下管理多本书。通过 `.novel/active-book` 指针实现。

## 执行流程

### 1. 发现所有书目

扫描项目根下包含以下关键子目录的目录：
- 包含 `正文/` 子目录
- 或包含 `设定集/` 子目录
- 或包含 `大纲/` 子目录

排除当前技能目录（novel/skills/）和隐藏目录。

### 2. 显示书目列表

```
📚 在当前项目中发现了以下书：

  1. 剑来           ← 当前活跃
  2. 暗卫
  3. 凡人修仙传

输入编号切换，回车取消。
```

### 3. 切换活跃书

用户选择后：
1. 读取 `.novel/active-book`（或 `.active-book` 兼容 story 系统）
2. 更新指针到所选书的相对路径
3. 更新 `.novel/state.json` 中的 project_info（读取新书的 state.json）
4. 输出切换确认

### 4. 查看当前活跃书

如果用户只问"我现在写哪本"不要求切换：
- 读取 active-book 文件
- 输出书名、当前进度（第 X 卷第 X 章）

### 5. 仅发现一本书时的行为

直接确认为活跃书，写入 active-book 文件，不询问用户。

## 参考

兼容 story-* 系统的 `.active-book` 格式。
首次切换如果 `.novel/active-book` 不存在，从上表发现的书目中让用户选择。
后续写作/查询/规划都自动指向活跃书目录。
novel-start 只针对当前活跃书执行。
novel-query 只查当前活跃书的数据，不跨书查询。
novel-export 只导出当前活跃书的内容。
